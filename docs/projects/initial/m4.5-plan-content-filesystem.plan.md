# Plan: Project-Scoped Specs & Full `spec` ‚Üí `project` Rename

## Overview
M4 left the UI in an inconsistent state and introduced a conceptual gap: "specs" and "projects" are the same thing but treated as separate concepts throughout the codebase. This plan does two things in one pass: (1) collapses the spec concept into project everywhere ‚Äî filesystem, service layer, tRPC API, components; (2) fixes the UI gaps ‚Äî removes the Default project card from the workspace overview, adds project creation, removes the workspace-level Specs tab, and adds a project-scoped Specs tab to each project page.

## Codebase Context

**Routing:**
- Workspace nav: `web/src/app/w/[workspace]/layout.tsx` ‚Äî tabs: Overview, Specs, Tasks, Docs, Memory
- Project layout: `web/src/app/w/[workspace]/projects/[project]/layout.tsx` ‚Äî tabs: Overview, Tasks, Plan, Diffs (disabled), PRs (disabled)

**Key files to rename/merge:**
- `web/src/server/spec/service.ts` ‚Äî `getSpecsDir()` returns `{workspace}/specs/`; all filesystem ops
- `web/src/server/trpc/routers/spec.ts` ‚Äî 12 tRPC procedures (`list`, `get`, `create`, `update`, `delete`, `readFile`, `writeFile`, `listContextFiles`, `readContextFile`, `writeContextFile`, `deleteContextFile`, `createProject`)
- `web/src/server/trpc/routers/project.ts` ‚Äî CRUD: `create`, `list`, `get`, `getBySlug`, `listWithProgress`, `updateStatus`, `delete`
- `web/src/components/specs/` ‚Äî `spec-tree.tsx`, `spec-frontmatter.tsx`, `spec-tasks.tsx`, `create-spec-dialog.tsx`
- `web/src/components/projects/create-project-dialog.tsx` ‚Äî creates project from spec (via `spec.createProject`; will be replaced)
- `web/src/server/db/schema.ts:39` ‚Äî `specPath: text('spec_path')` on projects table

**Filesystem path centralised in `spec/service.ts`:**
- `getSpecsDir(workspace)` ‚Üí `path.join(getWorkspaceDir(workspace), 'specs')`
- Also inlined in: `milestone.ts`, `plan-content.ts`, `mcp/index.ts` (all need updating)

**Other `specs/` references:** `engy-dir/init.ts` (workspace init), `ws/server.ts` (file watcher path check)

**5 frontend files using `trpc.spec.*`:** `specs/page.tsx`, `spec-tree.tsx`, `spec-frontmatter.tsx`, `create-spec-dialog.tsx`, `create-project-dialog.tsx`

## Functional Requirements

### Rename: `spec` ‚Üí `project` throughout
1. The filesystem directory for project content shall be `{workspace}/projects/{project-slug}/` (renamed from `specs/`). (user request)
2. On server startup, existing `{workspace}/specs/` directories shall be renamed to `{workspace}/projects/` automatically. (inferred: backwards compat for existing data)
3. The DB column `spec_path` shall be renamed to `project_dir` via a Drizzle migration. (user request)
4. All `spec.*` tRPC procedures shall be merged into the `project.*` router with new names per the rename map above. (user request)
5. `project.create` shall: auto-set `projectDir = slug`, insert the DB row, then initialise the filesystem directory (`projects/{slug}/` + `spec.md`). On filesystem failure, the DB row shall be deleted (compensating action). (inferred: follows existing workspace pattern)
6. `project.delete` shall: delete the DB row and remove the filesystem directory `projects/{projectDir}/`. (inferred: full cleanup)

### Workspace Overview
7. The Default project card shall NOT be displayed on the workspace overview. (user request)
8. The workspace overview shall display a "New Project" button that opens a dialog with a single name input. (user request)
9. On submit, the system shall call `project.create` and navigate to `/w/[workspace]/projects/[new-slug]/`. (inferred)

### Workspace Navigation
10. The Specs tab shall be removed from the workspace nav. (user request)
11. Navigating to `/w/[workspace]/specs` shall redirect to `/w/[workspace]`. (inferred: no broken URLs)

### Project Page ‚Äî Specs Tab
12. The project layout shall include a Specs tab at `/w/[workspace]/projects/[project]/specs`. (user request)
13. The Specs tab shall render `ProjectTree` + `ProjectDetail` (renamed SpecTree/SpecDetail) scoped to this project's directory. (user request)

## Non-Functional Requirements
- No new migration file ‚Äî edit existing migration + `schema.ts` in-place, delete `.dev-engy/engy.db`, restart (not in prod)
- Startup filesystem migration must be idempotent (skip if `projects/` already exists)
- All existing spec router tests must be migrated to the project router test file

## Workflow

**Creating a new project:**
1. Workspace overview ‚Üí click "New Project" ‚Üí dialog with name field
2. Submit ‚Üí `project.create` ‚Üí DB row inserted + `projects/{slug}/spec.md` created
3. Navigate to `/w/[workspace]/projects/[slug]/`
4. Specs tab available, shows project content editor

**Viewing/editing project spec:**
1. Project page ‚Üí Specs tab ‚Üí `ProjectTree` shows files in `projects/{slug}/`
2. Select file ‚Üí `ProjectDetail` editor renders content
3. Save ‚Üí `project.writeFile` persists to filesystem

## Out of Scope
- Changing the `/w/[workspace]/tasks` page or Tasks nav tab
- Memory, Docs, or other workspace-level tabs
- Removing the `projectDir` column (could be derived from slug, but keep for now)
- MCP spec tools being project-aware beyond field/path rename

## Implementation Sequence

1. **DB column rename** ‚Äî Edit `schema.ts`: rename `specPath: text('spec_path')` ‚Üí `projectDir: text('project_dir')`. Edit the existing migration file directly (same change). Delete `.dev-engy/engy.db` and restart ‚Äî server runs migrations on a fresh DB.

2. **Rename service layer** ‚Äî Move `server/spec/service.ts` ‚Üí `server/project/service.ts`. Rename `getSpecsDir` ‚Üí `getProjectsDir`, change `'specs'` ‚Üí `'projects'`. Move test file too.

3. **Filesystem migration** ‚Äî In `engy-dir/init.ts`: create `projects/` instead of `specs/`. Add startup check in `server.ts`: if `{workspace}/specs/` exists, rename to `projects/`.

4. **Update path references** ‚Äî In `milestone.ts`, `plan-content.ts`, `mcp/index.ts`, `ws/server.ts`: replace `'specs'` with `'projects'` and `specPath` with `projectDir`.

5. **Merge spec router into project router** ‚Äî Move all procedures from `spec.ts` into `project.ts` using the rename map. Remove `spec.create`, `spec.delete`, `spec.createProject`. Update `project.create` to also init filesystem. Update `project.delete` to also remove filesystem dir. Remove `spec.ts` file and its entry from `appRouter`.

6. **Migrate spec router tests** ‚Äî Move `spec.test.ts` tests into `project.test.ts`. Update all path/field references.

7. **Rename frontend components** ‚Äî Rename files in `components/specs/` ‚Üí `components/projects/` per the rename map. Update all imports.

8. **Update frontend tRPC calls** ‚Äî Update the 5 files using `trpc.spec.*` to use `trpc.project.*` with new procedure names.

9. **Remove workspace Specs route** ‚Äî Delete `app/w/[workspace]/specs/`. Add redirect page or Next.js redirect config.

10. **Remove Specs from workspace nav** ‚Äî `layout.tsx`: delete the Specs tab `<Link>`.

11. **Remove Default project card** ‚Äî `workspace-overview.tsx`: remove `isDefault` branch. Add "New Project" button ‚Üí new dialog component (name input only).

12. **Add Specs tab to project layout** ‚Äî `projects/[project]/layout.tsx`: add Specs tab entry.

13. **Create project specs page** ‚Äî `projects/[project]/specs/page.tsx`: render `ProjectTree` + `ProjectDetail` scoped to this project.

14. **Run `pnpm blt`** ‚Äî Verify build, lint, tests, knip, and jscpd all pass.
